interface、implements、extends以及泛型

interface

interface：接口，其作用与函数有点类似，函数规定了需要传递哪些参数，而接口则是规定某个对象应该拥有哪些属性或方法（只是声明成员方法，不作实现）。

// IBagView 和 IItemView 就是一个接口
interface IBagView {
    time: number;
    count?: number; //可选参数
    update(interval: number);
    refresh();
}

interface IItemView {
    updateItem(item, args);
}

implements

implements：实现，实现对应接口的对应方法或者环境变量，可以使用一个或者多个包含能的接口。实现了接口的类必须包含接口的所有成员方法。

class BagView implements IBagView, IItemView {
    public time = 0;
    public update(interval, number) {}
    public refresh():void  {}
    public updateItem(item: any) {}
    public update(item, args) {}
}

extends

extends：继承，继承的子类可以选择不需要重新定义类的结构及成员方法，如果子类重写了，则对象的方法或者变量会被子类覆盖。

class CommonBagView extends BagView
{
    constructor() {
        super();
    }

    public refresh():void  {
        //会覆盖父级的方法
        //可以使用super.refresh();进行继承
    }
}

泛型

泛型：泛型多被用来创建可重用的组件，一个组件可以支持多种数据类型。

假如不使用泛型

//这种数据类型就规定了必须使用Number类型
public getParam(param: number): number {
    return param;
}

//如果使用any，就无法准确知晓它的返回值类型
public getParam(param: any): any {
    return param;
}

使用泛型

//传入什么类型，返回值就是什么类型
public getParam<T>(param: T): T {
    return param;
}

泛型约束

//配合interface使用可以对泛型进行约束
//当然，被约束的泛型不再适用于任意类型
interface IBaseView {
    viewName: string;
    viewId: number;
}
//getView方法中进行了泛型约束
//那么调用这个方法时必须满足IBaseView的接口条件
public getView<T extends IBaseView>(view: T): T {
    return view;
}
