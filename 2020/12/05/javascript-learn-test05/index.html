worker的使用。

worket简单介绍

JavaScript 语言采用的是单线程模型，也就是说，所有任务只能在一个线程上完成，一次只能做一件事。前面的任务没做完，后面的任务只能等着。而Web Worker 的作用，就是为 JavaScript 创造多线程环境，允许主线程创建 Worker 线程，将一些任务分配给后者运行。在主线程运行的同时，Worker 线程在后台运行，两者互不干扰。等到 Worker 线程完成计算任务，再把结果返回给主线程。

好处：一些计算密集型或高延迟的任务，被 Worker 线程负担了，主线程（通常负责 UI 交互）就会很流畅，不会被阻塞或拖慢。

worker简单使用

注意：

  1. 同源限制，分配给 Worker 线程运行的脚本文件，必须与主线程的脚本文件同源
  2. Worker无法访问DOM对象，例如document、window等；但可以访问navigator、location对象
  3. 主线程与子线程的通信方式，必须通过消息联系，而不能通过上下文环境控制
  4. 一些方法脚本的限制，Worker无法使用alert()和confim()方法
  5. 文件限制，Worker脚本文件必须来源于网络，而不能是本地
  6. Worker 比较耗费资源，不应该过度使用，而且一旦使用完毕，就应该关闭
//main.js
let worker = new worker() //网络脚本文件，注意：Worker不能读取本地文件
let date = new Date();
let dateStr = date.getFullYear();
dateStr += "-";
dateStr += date.getMonth() + 1 > 9 ? date.getMonth() + 1 : "0" + (date.getMonth() + 1);
dateStr += "-";
dateStr += date.getDate() > 9 ? date.getDate() : "0" + date.getDate();
dateStr += ":";
dateStr += date.getHours() > 9 ? date.getHours() : "0" + date.getHours();
dateStr += " ";
dateStr += date.getMinutes() > 9 ? date.getMinutes() : "0" + date.getMinutes();
dateStr += ":";
dateStr += date.getSeconds() > 9 ? date.getSeconds() : "0" + date.getSeconds();
worker.postMessage("begin", dateStr) //主线程向子线程发送消息
worker.onmessage = (e) => { //主线程监听子线程消息
    doSomething(e.data);
}

function doSomething(data) {
    console.log("onmessage by worker Data :", data)
    worker.terminate(); //主线程关闭worker
}
//如果需要加载外部脚本，使用 self.importScripts(jsfile) 方法

//work.js
function addCount() {
    let count = 0;
    return () => {
        count += 1;
    }
}

function beginAddFunc() {
    let count = addCount();
    let timer = setTimeout(() => {
        let val = count();
        if (val == 10) {
            self.postMessage("baginOver:" +  val); //给主线程发送消息
            self.close(); //关闭自身
            clearTimeout(timer);
        }
    }, 1000);
}

self.addEventListener("begin", (e) => { //监听主线程消息
    console.log("beginEvent Data:", e.data);
    self.beginAddFunc(); 
}, false)

错误处理

主线程可以通过worker的error事件来监听worker的错误。

worker.onerror((e) => {
    //some code ...
})
//或者
worket.addEventListener('error', (e) => {
    //some code ...
})

数据通信

主线程与Worker之间通信时，通信内容一般是文本、对象，这种通信是拷贝关系，即Worker对通信内容的修改，不会影响到主进程。这里有个特殊的地方，假如主线程向Worker发送一个略大的二进制文件时，为了防止出现多个线程同时修改数据的问题，用了一种转移的方法，使得主线程可以把数据控制权转移给Worker（这个时候主线程已经无法再使用这些数据了，已经转移到Worker）。

//Transferable Objects 格式
worker.postMessage(arrayBuffer, [arrayBuffer]);

//example
var ab = new ArrayBuffer(1);
worker.postMessage(ab, [ab]);
